// why nxv

Stop spelunking through commits.

nxv has indexed 9+ years of nixpkgs history so you don't have to. Query it any way you like.

Blazingly fast

A Bloom filter returns instant "not found" responses; SQLite FTS5 powers full-text search across millions of versions.

Complete history

Every channel-released nixpkgs version since 2016 — including nested sets like python3Packages — at cache-backed commits.

Offline-first

Download the ~220MB index once and query locally — no network required after the initial sync.

Three interfaces

A ~10MB static CLI binary, an HTTP API server with a web UI, or query a remote API transparently.

Security aware

Clear indicators for insecure packages and known vulnerabilities, with an opt-in toggle to include them.

Agent-ready

Ships an Agent Skills-standard skill so Claude, Codex, and other coding agents can drive nxv for you.

// how it works

From channel snapshot to nix shell.

pipeline
  channel-release snapshots        indexer                  SQLite index
  releases.nixos.org  ──▶  packages.json.br  ──▶  + bloom filter
       (S3, back to 2016)          (streamed parse)              (~220MB)
                                                                    │
                                                                    ▼
  nxv search / nxv serve  ◀──  query engine  ◀──  download from remote/local
  # every recorded commit is one where the version verifiably existed

Install in one command.

Grab a pre-built static binary, install via Cargo, or run it straight from the flake — no index build required to query the live API.

$ curl -sSfL https://raw.githubusercontent.com/utensils/nxv/main/install.sh | sh copy
$ cargo install nxv copy
$ nix run github:utensils/nxv -- search python copy
